<span style="color:rgb(0, 176, 80)">Zanke</span> so način, kako v programskih jezikih efektivno izvajamo nek del kode večkrat.

V Pythonu sta dve glavni zanki: <span style="color:rgb(0, 176, 240)">while</span> zanka in <span style="color:rgb(0, 176, 240)">for</span> zanka.

#### While zanka

Za while zanko se stvari izvajajo, ko je ta pogoj, ki ga podamo, izpolnjen (njegova vrednost je True).
Pogoj je lahko nekaj statičnega, lahko je kaj dinamičnega (da se spreminja). Za while zanko najpogosteje podamo tudi nek ustavitveni pogoj v tem pogoju, da se to ne ponavlja v nedogled.

```python
def narisi_zvezdice(n):
	while True: # tole npr. bo vedno res in se bo vedno ponavljalo - statično
		print(n * "*")
		
def narisi_pikice(n):
	stevec = 0
	while stevec < n:
		print(n * ".")
		stevec += 1 # vedno ko se zanka izvede (vsak cikel), se preveri začetni pogoj, prej ali slej bo stevec enak n in while zanka se bo nehala izvajati
```

Primer: harmonična vrsta $\displaystyle{\sum_{n=1}^{\infty} \frac{1}{n}}$ divergira. Presegla bo katerokoli pozitivno vrednost.


Definirajmo funkcijo, ki pogleda, koliko členov je potrebno, da harmonična vrsta preseže neko vrednost x:

```python
def clen_preseze(x):
	vsota = 1
	n = 2
	
	while vsota <= x:
		vsota += 1/n
		n += 1
	
	return n - 1 # zmanjšamo za 1, ker se je povečalo za 1 po cilju
```
___
Opomba:
1. Tipična napaka v Pythonu je off by one. Pogosto se kaj poveča/zmanjša za 1, ko mi tega ne želimo. To se hitro popravi s tem, da samo odštejemo/prištejemo 1. Pogosto je dobro preveriti, ali funkcija deluje OK na manjših primerih ali smo off by one.

#### For zanka

For zanka je zelo koristna v Pythonu. Motivacija za uporabo for zank je, da nam ni treba skrbeti za indekse in delanje z while zankami. 

```python
def stevilo_samoglasnikov(niz):
	stevilo = 0
	for crka in niz: # šlo bo čez vse črke v nizu
		if crka.lower() in "aeiou": # preveri, ali je črka samoglasnik
			stevilo += 1
	
	return stevilo
```

Python ima vgrajeno funkcijo, ki se imenuje <span style="color:rgb(0, 176, 240)">range</span>, ki se uporablja pri for zankah. Kar dela je, da teče od začetnega števila (če nič ne damo je enako 0) do števila, ki ga podamo (ne vključuje tega števila):

```python
def zvezdice2(n):
	for i in range(10): # i bo šel od 0 do 9, ne bo pa 10
		print(i * "*")

def zvezdice3(n):
	for j in range(1, 15, 2): # argumenti so začetek, konec, korak
		print(j * "*")
		
def stejem_nazaj(n):
	for k in range(n, 0, -1): # korak je 1 nazaj
		print(k)
```

Primer: Fakulteto se da izračunati tudi s for zanko:

```python
def fakulteta(n):
	produkt = 1
	for i in range(1, n+1): # to bo šlo od 1 do n
		produkt *= i
		
	return produkt
```

Imamo še dva pomembna ukaza: <span style="color:rgb(0, 176, 240)">continue</span> in <span style="color:rgb(0, 176, 240)">break</span>. Break prisili, da "skočimo" ven iz zanke oziroma, da nehamo izvajati zanko, continue pa preskoči samo en specifičen cikel zanke.

```python
def zvezdice4(n):
	for i in range(n):
		if i >= 10:
			break # prisili, da se zanka konča, ne konča pa cele funkcije!
		if i == 3:
			continue # preskoči ta cikel zanke in gre v naslednjega
		print(i * "*")
```

Primer: lahko pogledamo, ali je nek podniz znotraj niza:

```python
def je_podniz(podniz, niz):
	for i in range(len(niz) - len(podniz)): # sprehodili se bomo po indeksih niza
		sta_enaki = True
		for j in range(len(podniz)): # in po indeksih podniza
			if niz[i + j] != podniz[j]: # hkrati moramo zamakniti obe
				sta_enaki = False
				break
		if sta_enaki:
			return True
	return False
```

Ta funkcija je dokaj neučinkovita ker imamo dve for zanki eno znotraj druge. V big O notaciji (časovna notacija za ocenitev časa, ki ga porabi program), je ta program O($n^2$).

Primeri:
1. Razvijmo zelo preprosto metodo za oceniti število $\pi$. To naredimo s tem, da vzamemo 2x2 kvadrat, notri včrtamo krog, naključno postavimo točke v kvadrat in pogledamo, koliko od teh je znotraj kroga. Realna verjetnost, da je točka notri v krogu je $\frac{\pi}{4}$, ker je ploščina kroga enaka $\pi$, ploščina kvadrata pa 4. V Pythonu bomo izvedli to simulacijo.

```python
import random # za naključne točke

def oceni_pi(n):
	stevec = 0
	for _ in range(n): # lahko damo namesto spremenljivke podčrtaj, če je ne rabimo - konvencija v Pythonu
		x = random.random() # ta funkcija vrne vrednost med 0 in 1
		x = 2 * x - 1 # zdaj je enakomerno naključno med -1 in 1 - za 2x2 kvadrat
		y = random.random() # še y koordinata točke
		y = 2 * y - 1
		
		if x**2 + y**2 <= 1: # preverimo, če je točka v krogu
			stevec += 1 
			
	return 4 * stevec / n # vrne pi
```

3. Zamislimo si, da smo v eni dimenziji in se v vsakem koraku odločimo, ali gremo 1 levo ali 1 desno. Ali bomo prišli nazaj na izhodišče? Kakšna je verjetnost, da pridemo nazaj v izhodišče? Bomo naredili simulacijo:

```python
def nakljucni_sprehod(n, m):
	pozicija = 0
	stevec = 0
	for _ in range(n): # n - krat naredimo simulacijo
		for i in range(m): # m bo zelo velika, da simulira 
			korak = random.choice([-1,1]) # bo izbral eno število, -1 ali 1
			pozicija += korak
			if pozicija == 0:
				stevec += 1
				break # ko pride nazaj do 0 bomo šli ven iz te notranje zanke
	return stevec / n # to nam bo dalo delež kolikokrat pride nazaj
```

Izkaže se, da je verjetnost, da pridemo nazaj enaka 1, verjetnost pa da nekam odbluzimo je 0.
